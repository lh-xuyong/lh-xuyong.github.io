Lravel表单验证规则

Lravel表单验证规则

accepted

验证的字段必须为yes on 1 true . 一般用于服务条款是否同意

active_url

相当于使用了PHP函数 dns_get_record,验证字段必须具有有效的A或AAAA

记录,即checkdnsrr – 根据一个给定的host name(域名)或者IP地址检查它是否有DNS记录，其实也就是检验它是否存在

after:date

验证的字段必须是给定日期后的值,这个日期将会通过PHP函数strtotime验证

你也可以指定其他的字段来比较日期

ater_or_equal:date

验证的字段必须等于给定日期或者之后的值

alpha

验证的字段完全是字母的字段

alpha_num

验证的字段必须完全是是字母丶数字

array

验证的字段必须是一个PHP数组

before:date before_or_equal:date

和上面after相反

between:min,max

字段的大小在给定之间

boolean

验证的字段必须能被转换为布尔值

confirmed

验证的字段必须和 foo_confirmation的字段一直,例如要验证的字段是password,输入中必须存在陪陪的password_confirmation

date

能通过strtotime校验的有效日期

date_equals:date

验证的字段必须等于给定的日期strtotime

date_format:format

验证字段必须与给定的格式相匹配,date和date_format只能用一个

different:field

验证的字段值必须与字段值不同

digits:value

验证的字段必须是数字,并且必须具有确切的值

numeric

验证的字段必须是数字

digits_between:min,max

验证的字段长度必须在给定的min和max之间

dimensions

验证的文件必须是图片并且图片必须符合规则:

distinct

验证数组时,指定的字段不能有任何重复值

email

验证的字段必须符合e-mail的地址格式

exists:table,column

验证的字段必须存在于给定的数据库表中

file

字段必须是成功上传的文件

fiiied

验证的字段在存在时不能为空

image

验证的文件是一个图像

in:foo,bar,…

not_in:foo,bar,…

验证的字段必须包含在/不在给定的值列表中,要用到in,就像sqlin查询一样

in_array:anotherfield

验证的字段必须存在于另一个字段的值中

integer

验证的字段必须时整数

ip ipv4 ipv6 json

max:value min:value 验证中的字段必须具有最大/小值

mimetypes:text/plain,…

mimes:foo,bar,…

验证的文件必须具有与列出的其中一个扩展名相应的MIME类型

MIME规则的基本用法

nullable

验证的字段可以为null 验证基本数据类型时特别有用,例如可以包含空值的字符串和整数

present

验证的字段必须存在于数据中,但可以为空

regex:pattern

验证的字段必须与给定的正则表达式匹配,使用rgex规则时必须使用数组,而不是|分隔符,因为正则里含有|

required

string

验证的字段必须是字符串,如果允许该字段的值为null,九江nullable规则加到该字段

timezone

判单字段是有效的失去标识,以timezone_identifiers_list来判断

unique:table,column,except,idColumn

验证的字段在给定的数据库表中必须是唯一的,如果没有指定的colum,将会使用字段本名称

url

验证的字段必须是有效的 URL。
