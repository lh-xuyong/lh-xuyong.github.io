CSRF和XSS的攻击与防御…

XSS

Cross Site Scripting:跨站脚本,X代表未知的含义

跨站攻击

用户->目标站点->渲染HTML过程中出现没有预期的脚本->攻击者通过脚本获取到用户的信息(如cookie)

危害

  • 挂马
  • 盗取用户cookie
  • DDOS(拒绝服务)客户端浏览器
  • 钓鱼攻击,高级的钓鱼技巧
  • 劫持用户Web行为
  • 蠕虫病毒

类型

  • 反射型: URL注入非法脚本->服务器解析响应->反射给受害用户 (一般浏览器都会有防护),这种攻击是非持久性的(一般被用于邮箱)
  • 存储型: 与反射型的区别就是提交的URL会存储在服务器数据库,(一般被用于留言)
  • DOM-Based型: 基于DOM或本地的XSS攻击

防御

输入

对输入(和URL参数)进行过滤,对输出进行编码.

  • 用户输入丶RUL参数丶POST请求参数丶Ajax
  • 黑白名单
输出
  • 对输出代码转义
    cookie
    cookie,服务端设置为http-only,js不能获取到cookie了

CSRF

csrf 跨站请求伪造:攻击者利用了用户的登录的状态,以用户的名义发送恶意请求. 根本原因在Web服务器验证不够(只是验证了session的存在,无法验证是否是用户触发的)

用户登录了一个正常访问网站(有CSRF漏洞)->session存在->在正常网站没退出的情况下访问了恶意网站->恶意网站的图片(例:图片资源)->网站图片连接里嵌入了接口带上恶意参数(get请求下)

防御

  • 尽量使用POST
  • 加入验证码
  • 验证Referer(原理:站点不同)
  • Anti CSRF Token : 1.head或form表单传递token 2.token存储在服务端 3.服务端通过拦截器验证有效性 4.效验失败拒绝请求
可以打赏,但没必要
