小甲鱼

数据结构

学术:数据结构是一门研究非数值计算的程序设计问题对象,以及它们之间的关系和操作等相关问题的学科.

小甲鱼:

  • 程序设计 = 数据结构+算法
  • 数据结构就是关系,数据元素相互之间存在的一种或多种特定关系的集合
  • 数据结构分为逻辑结构和物理结构

逻辑结构

数据对象中数据元素之间的相互关系,也是主要学习的

集合结构

集合结构中的数据元素除了同属于一个集合外,他们之间没有其他的不三不四的关系

树形结构

树形结构中的数据元素之间存在一种一对多的层次关系(二叉树?)

图形结构

图形结构的数据元素是多对的关系(杂交)

(理解:6度理论,我可以通过6个人间接认识世界上所有的人)

物理结构

数据的逻辑结构在计算机中的存储形式

顺序存储

把数据元素存放在地址连续的存储单元里,其数据间的逻辑关系和物理关系是一致的

例如:数组

链式存储

顺序存储(如医院就医排队领号你只能站着队伍排队), 时常要变化的机构,顺序存储就是不科学的,那么就要用到链式存储了

链式存储(医院见医生时叫号系统,你随便站在那儿都行,到号时就医就行)

这里的你就代表数据元素

算法

数据结构和算法的关系:好基友一辈子

学术:算法是指解决特定问题求解步骤的描述,在计算机中表现为指令的优先序列,并且每条指令表示一个或多个操作

小甲鱼: 算法就是你泡妞的技巧和方式

算法的5个基本特征

输入

算法具有零个或多个输入

输出

算法至少有一个或多个输出

有穷性

算法不会无限循环,算法就是为了得到输出,一个永远追不到的妹子你还追她有何用

确定性

算法每一个步骤都具有确定的含义,不会出现二义性,爱就是爱不爱就是不爱

算法在一定条件下,只有一条执行路径,相同的输入只能有唯一的输出结果

算法的每个不走都应该被精确定义而无歧义

可行性

算法的每一步都是可行的,也就是说每一步都能通过执行有限次数完成

算法设计的要求

算法不是唯一,同一个问题可以有多种算法解决

尽管算法不唯一,还是学一些算法可以解决很多问题

正确性
  • 算法程序无语法错误
  • 算法程序对于合法输入能够产生满足要求的输出
  • 算法程序对于非法输入能够产生 满足规格的说明
  • 算法程序对于故意刁难的测试输入都有满足要求的输出结果(难)
可读性

算法另一目的是为了便于阅读理解和交流或修改

健壮性

输入数据不合法时不能异常崩溃或者跳出一些乱七八糟的东西

时间效率高和存储量低

好的算法像老婆,温柔体贴美丽大方,一手好菜

算法时间复杂度

执行次数 == 时间

大O记法:用大写0()来体现算法时间复杂度记法
